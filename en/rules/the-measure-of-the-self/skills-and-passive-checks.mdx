# Skills and Passive Checks

## Overview

This rule describes how skill proficiencies apply to ability checks, which skills are tied to which ability scores, how a GM may apply a skill to an unusual ability check, and how passive checks work.

## Mechanics

- Proficiency bonus: When a character is proficient in a skill, add the character's proficiency bonus to ability checks that involve that skill. A monster's skill proficiencies appear in its stat block.
- Skill proficiency is determined at character creation for PCs; for monsters it is listed in the stat block.
- A skill normally applies only to the specific kind of ability check associated with it. For example, proficiency in Athletics normally applies to Strength checks that use Athletics. When you make an ability check that involves a skill you are proficient in, add your proficiency bonus to the ability check.

Skills associated with each ability score:

- Strength
  - Athletics

- Dexterity
  - Acrobatics
  - Sleight of Hand
  - Stealth

- Intelligence
  - Arcana
  - History
  - Investigation
  - Nature
  - Religion

- Wisdom
  - Animal Handling
  - Insight
  - Medicine
  - Perception
  - Survival

- Charisma
  - Deception
  - Intimidation
  - Performance
  - Persuasion

## Variant: Skills with Different Abilities

- The GM may reasonably call for a check that mixes a skill with a different ability than usual when the situation calls for it. In such cases, the proficiency applies to that unusual combination.
- Examples:
  - If a situation requires stamina rather than raw strength (for example, swimming a long distance from an offshore island to the mainland), the GM might call for a Constitution (Athletics) check. If you are proficient in Athletics, you apply your proficiency bonus to the Constitution (Athletics) check just as you would to a Strength (Athletics) check.
  - If a character attempts an acrobatic stunt to palm an object or stay hidden, the GM might ask for a Dexterity (Acrobatics) or Dexterity (Sleight of Hand) check.
  - If your half-orc barbarian uses a display of raw strength to intimidate an enemy, the GM might ask for a Strength (Intimidation) check even though Intimidation is normally associated with Charisma.

- Rule: When the GM specifies a skill with a different ability, treat the proficiency the same way you would for the skill's normal ability: add the proficiency bonus if proficient.

## Passive Checks

- A passive check represents the average result for a task done repeatedly or the result used when the GM wants to determine success without rolling dice (for example, quietly determining whether the characters notice a hidden creature).

- Passive check total formula:

  10 + all modifiers that normally apply to the check.

  This includes ability modifiers, proficiency bonus (if the character is proficient in the relevant skill), and any other modifiers that would normally apply.

- Adjustment for advantage and disadvantage on passive checks:
  - If the character has advantage on the check, add 5 to the passive total.
  - If the character has disadvantage on the check, subtract 5 from the passive total.

- Examples and uses:
  - Use a passive Perception to determine whether a character notices a hidden monster when the GM does not want to call for an active roll.
  - Use passive checks to represent the typical outcome of repeated actions such as searching the same room over and over.
  - If the GM wants a specific skill to apply, the GM might ask for a "Wisdom (Perception) check." If the character has proficiency in the named skill, apply the proficiency bonus as normal.

## Interactions

- Skill proficiencies interact with ability checks: proficiency applies only when the check uses the skill the character is proficient in (or the GM allows the skill to apply to an unusual ability check as described above).
- Monster stat blocks list any skill proficiencies a monster has; apply the listed bonuses to the monster's ability checks for those skills.
- Passive checks are a deterministic substitute for rolling in situations where an average or repeated outcome is appropriate; they obey the same modifiers as active checks, including proficiency and situational modifiers.
